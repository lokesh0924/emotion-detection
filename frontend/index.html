    How this system works

    End‑to‑end pipeline

    1. Pre‑processing: emojis are converted to text, punctuation and numbers are removed, the text is tokenized, stopwords are removed, and the remaining tokens are lemmatized.
    2. Feature extraction: we apply a Count Vectorizer followed by TF‑IDF to represent each text as a numeric vector.
    3. Model inference: six different machine‑learning models are queried with the same vector.
    4. Aggregation: the web app combines their outputs into a final, more stable prediction.

    Models behind the scene

    SVM (SGDClassifier)

    Linear Support Vector Machine trained with stochastic gradient descent. This model achieved the highest accuracy in your experiments (~89.5%).

    Logistic Regression

    Multiclass logistic regression used as a strong linear baseline for text classification tasks.

    Random Forest

    An ensemble of decision trees. More robust than a single tree and able to capture non‑linear feature interactions.

    XGBoost

    Gradient‑boosted tree ensemble, often among the best performing algorithms on tabular/text features.

    Multinomial Naive Bayes

    Classic probabilistic model for text that works surprisingly well for bag‑of‑words features.

    Decision Tree

    A single interpretable tree that splits on TF‑IDF features to reach emotion predictions.
